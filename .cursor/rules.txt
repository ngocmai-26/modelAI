# Project Context – Machine Learning Library for CLO Prediction & XAI

## 1. Project Overview

This project is a Python-based Machine Learning library designed to:

- Predict CLO scores (scale 0–6) using regression models
- Perform statistical analysis and correlation analysis
- Explain predictions using XAI (SHAP-based explanations)
- Generate educational reasons and actionable solutions
- Support both individual student analysis and class-level analysis
- Be imported and used inside backend services (NOT a standalone app)

**CRITICAL**: The project is NOT a notebook-based experiment repository.
All code must be production-ready, testable, and reusable as a library.

Scientific background, data assumptions, and model logic are defined in `.cursor/summary.txt`.

**Documentation**: All model documentation, requirements, and data model specifications are located in the `docs/` directory. Key documents include:
- `docs/model_requirements.md`: Model requirements and specifications
- `docs/data_model.md`: Data model definitions and structure

## 2. Technology Stack

- **Language**: Python >= 3.10
- **ML stack**:
  - pandas, numpy
  - scikit-learn
  - shap (TreeExplainer only)
  - matplotlib / seaborn (visualization only, optional)
- **Serialization**: joblib or pickle for models
- **Testing**: pytest
- **Build**: pyproject.toml, setuptools

**Forbidden**:
- No deep learning frameworks unless explicitly requested
- No notebook (.ipynb) in core library code

## 4. Data & Scientific Assumptions

### 4.1 Score Scale Conversion (CRITICAL)

- **All CLO scores are on a 0–6 scale**
- **Historical exam scores (0–10) MUST be converted before training**
- **Conversion formula**: `CLO_6 = Score_10 / 10 × 6`
- **No model ever sees raw 0–10 scores**
- All training data must be in 0–6 scale
- Future data from system will already be in 0–6 scale (no conversion needed)

### 4.2 Data Sources

The library must handle these data sources:
- **Điểm rèn luyện** (Conduct scores): DRL_2021_2024.xlsx
- **Điểm tổng** (Exam scores): Diem_Tong.xlsx
- **Nhân khẩu học** (Demographics): Data K24 25 26 27.xlsx
- **Phương pháp đánh giá** (Assessment methods): PPDG.xlsx / PPDGfull.xlsx
- **Phương pháp giảng dạy** (Teaching methods): PPGD.xlsx / PPGDfull.xlsx
- **Giờ thư viện** (Library hours): Gio_TV_22-24.xlsx
- **Điểm danh** (Attendance): (to be specified)

### 4.3 Data Preprocessing Rules

All preprocessing steps must be:
- **Deterministic**: Same input → same output
- **Reproducible**: Fixed random seeds
- **Isolated in `data/`**: No preprocessing logic outside this module
- **Validated**: Use `validators.py` to check data quality

**Specific preprocessing requirements**:
- Standardize `Student_ID`, `Subject_ID`, `Lecturer_ID`
- Clean `exam_score`: remove invalid values, convert to float
- Encode teaching methods: `X → 1`, others/NaN → 0
- Encode demographics:
  - Gender: binary encoding
  - Birth place → region (Bắc/Trung/Nam)
  - Ethnicity, religion → integer encoding
- Create binary `Result` variable: Pass (≥ 6), Fail (< 6) [for analysis only, not for training]

### 4.4 Feature Engineering

Features must be organized into these groups (defined in `feature_groups.py`):

**A. Basic Information**:
- `student_id_encoded`, `lecturer_encoded`, `subject_encoded`

**B. Demographics**:
- `gender_encoded`, `religion_encoded`, `birth_place_encoded`, `ethnicity_encoded`

**C. Conduct Scores**:
- `avg_conduct_score`, `latest_conduct_score`, `conduct_trend`

**D. Self-Study**:
- `study_hours_this_year`, `total_study_hours`

**E. Academic History**:
- `total_subjects`, `passed_subjects`, `pass_rate`
- `avg_exam_score`, `recent_avg_score`, `improvement_trend`

**F. Teaching & Assessment Methods**:
- Teaching method features (from PPGDfull.xlsx)
- Assessment method features (from PPDGfull.xlsx)

## 5. Modeling Rules

### 5.1 Problem Type
- **Regression problem** (not classification)
- **Target variable**: CLO score (0–6, continuous)

### 5.2 Allowed Models

**Random Forest Regressor**:
- `n_estimators = 1000`
- `max_depth = 25`
- `random_state = 42`

**Gradient Boosting Regressor**:
- `n_estimators = 500`
- `max_depth = 12`
- `learning_rate = 0.03`
- `random_state = 42`

**Ensemble**:
- Weighted average of both models
- Weights determined by validation performance

### 5.3 Model Interface

Each model must:
- Inherit from `models.base.BaseModel`
- Expose: `train()`, `predict()`, `save()`, `load()`
- Be serializable using joblib or pickle
- Support model versioning
- Log training metrics

### 5.4 Training Rules

- **No training logic in backend runtime paths**
- Training only happens in `scripts/train.py`
- Full retraining (no incremental learning)
- Model versioning is mandatory
- Log training data size and metrics

## 6. XAI & Explainability Rules

### 6.1 SHAP Configuration

- **SHAP is the ONLY explainability method**
- Use `shap.TreeExplainer` for tree-based models
- **Raw SHAP values must NOT be exposed directly**
- Must be processed through `xai/shap_postprocess.py`

### 6.2 SHAP Processing Flow

1. **Compute SHAP values** using TreeExplainer
2. **Filter by absolute threshold** (configurable in `xai_config.py`)
3. **Group by pedagogical feature groups**:
   - **Tự học** (Self-study): Library hours
   - **Chuyên cần** (Attendance): Attendance rate
   - **Rèn luyện** (Conduct): Conduct scores
   - **Học lực** (Academic): Exam scores
   - **Giảng dạy** (Teaching): Teaching methods
   - **Đánh giá** (Assessment): Assessment methods
   - **Cá nhân** (Personal): Demographics
4. **Normalize to percentage impact** for interpretation

### 6.3 Individual Analysis (XAI)

For each student:
- Compute SHAP for all features
- Identify negative SHAP values (reducing score)
- Group by pedagogical categories
- Calculate impact percentage per group
- Return top N reasons with highest negative impact

### 6.4 Class Analysis (XAI)

For class-level:
- Compute SHAP for all students
- Average SHAP values per feature across class
- Group by pedagogical categories
- Calculate average impact percentage per group
- Return top N common reasons

### 6.4 Reason Generation

- **Must be rule-based** (no ML, no LLMs)
- Map from XAI outputs to pedagogical reasons
- Use templates from `reasoning/templates.py`
- Generate human-readable, educational language
- Focus on: learning behavior, participation level, study methods

### 6.5 Solution Mapping

- **Deterministic and template-driven**
- Map reason keys (e.g., `LOW_SELF_STUDY`, `LOW_ATTENDANCE`) to solutions
- Context-aware (individual vs class)
- Solutions must be:
  - Easy to understand
  - Actionable
  - Appropriate for university education context

## 8. Data Storage & Retraining

- **MUST store actual CLO scores** (0–6 scale) for future training
- Store: `student_id`, `subject_id`, `lecturer_id`, `clo_score`, features, semester/year
- Purpose: Use as training data for future retraining

### 8.2 Retraining Strategy
