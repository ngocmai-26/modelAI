## 14. CÁC ĐIỂM QUAN TRỌNG CẦN LƯU Ý

### 14.1 Score Conversion (CRITICAL)
- **TẤT CẢ điểm thi hệ 10 PHẢI chuyển sang hệ 6 trước khi training**
- Công thức: `CLO_6 = Score_10 / 10 × 6`
- Model chỉ nhìn thấy điểm hệ 6
- Dữ liệu tương lai từ hệ thống sẽ là hệ 6 (không cần convert)

### 14.2 Data Storage for Retraining
- Khi phân tích lớp, **PHẢI lưu lại điểm CLO thực tế** (hệ 6)
- Lưu: student_id, subject_id, lecturer_id, clo_score, features, semester/year
- Dùng cho retraining trong tương lai

### 14.3 Model Versioning
- Mỗi model phải có version (v{MAJOR}.{MINOR}.{PATCH})
- Lưu metadata với model
- Support load model by version

### 14.4 Reproducibility
- Fixed random seeds (random_state=42)
- Deterministic preprocessing
- Log all random seeds used

### 14.5 No Forbidden Actions
- ❌ Không dùng notebook (.ipynb) trong core code
- ❌ Không dùng LLM cho reason generation
- ❌ Không có print() hoặc input()
- ❌ Không hard-code paths hoặc credentials
- ❌ Không skip error handling

### 14.6 Code Quality
- Type hints cho tất cả functions
- Docstrings (Google style) cho public APIs
- PEP 8 compliance
- Maximum line length: 100 characters
